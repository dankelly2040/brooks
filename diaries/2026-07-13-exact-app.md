# Exact app: the Brooks storefront in Contract, built and agent-verified

**Date:** 2026-07-13
**Agent:** Claude Fable 5
**System:** Exact
**Scope:** Built `apps/exact` — the full Brooks shopping journey in Contract on
a fresh clone of `origin/main`, verified end-to-end through the agent API.

## Outcome

A working Brooks storefront in one `.contract` route (`~620 lines`) plus a pure
data module: home (with a **live ticking Project 222 countdown**), browse with
category chips and client-side search, product detail with colorway swatches,
gallery, width/size selection gated add-to-bag, and a cart that builds real
Brooks variant ids — same catalog snapshot, same live Brooks CDN photography as
the Expo app.

Verified through Acto (the agent API), not hand-driving: 15 steps — countdown
tick observed across snapshots, browse → chip filter → search narrows 61→7 →
detail → add gated until size → width+size → variant
`1105302E421.080` previewed → add ×2 → badge 2 → colorway swap resets size →
cart totals ($239.90, free shipping) → qty stepper → checkout scope note →
remove → empty state. **All pass, zero console errors.** Screenshots in
`docs/exact-*.png`.

**Checkout decision (LLP 0000 open question):** built against a **fresh clone
of `origin/main` at `~/projects/exact-main` (c3f49e50)**, per Charlie's explicit
call. The dirty tree at `~/projects/exact` was not touched. `exact.links.json`
records the linkage.

## What worked well

- **`exact new` is a real paved road.** Scaffold → `bun install` →
  `bun --bun run dev` → rendering Contract at `:8083` in under two minutes
  (after two one-time repo steps below).
- **`temporal()` carries the countdown.** LLP 0004 said "no animation, no
  ticking UI" — that was wrong in the best way. A `text temporal(() => ({value,
  nextChangeAt}))` binding gives a once-per-second countdown that doesn't wake
  app derives. This is the single feature I expected to lose vs Expo and didn't.
- **The pure-helper + `.contract-meta.ts` discipline is genuinely good for
  agents.** Being forced to put every data access behind a declared-pure helper
  made the reference-stability rules (Object.is propagation) explicit instead
  of folklore. `memoRows("id")` on the search-filtered grid worked first try.
- **Acto is a better verification loop than Playwright selectors.** testIds in
  the contract are simultaneously the spec, the agent handle, and the test
  hook. The whole journey was driven by `tap {testId}` + snapshot reads.
- **The Brooks CDN + bundled snapshot architecture (LLP 0002) ported verbatim.**
  Same bare URLs, same `?sw=&sh=` sizing, zero data-layer surprises.

## Friction and blockers

All on `origin/main` (c3f49e50), all reproducible:

1. **Fresh clones don't build without spelunking.** `cargo build -p exact`
   fails (the crate is `exact-cli`), and before that the build fails outright
   until `git submodule update --init vendor/ibex`. Neither step is in the
   getting-started path.
2. **The in-browser agent server is broken in plain browser tabs.**
   `runtime-bootstrap` → `server.ts` → code-mode → `node:worker_threads` and
   `node:crypto` at module scope. Vite externalizes node builtins into
   throw-on-access shims, so the whole agent bootstrap dies —
   `installContractWebHost` apps get `NO_REGISTERED_TARGET`. Worked around
   with two Vite aliases (`src/shims/worker-threads.ts`,
   `src/shims/node-crypto.ts` — the latter needs a *working* sha256 because
   code-mode hashes its SDK manifest at import time). The web target then
   registers and the full agent surface works.
3. **`bun run typecheck` in a scaffolded app fails out of the box** — ~20
   errors, every one inside the source-linked `@exact/*` packages themselves
   (exact-router, exact-devtools), none in app code. The scaffold's tsc
   settings are stricter than the packages it links.
4. **Snapshots taken during re-render windows return `refs: {}`**, and
   tap-by-testId 422s against them. Any driving script needs retry/poll
   discipline (mine: 4×500ms on tap, poll-until-present on reads). `exact_wait`
   likely covers this; the raw HTTP surface makes it easy to hit.
5. **Multi-target routing bit hard.** The relay routes to "most recently
   active" target by default; stale targets from dead pages (crashed test
   browsers, and one `mac-*` target) linger in `/agent/targets`, and my taps
   silently drove a zombie page while screenshots watched the live one. Fix:
   read the page's own id from `sessionStorage.__exact_web_agent_target_id`
   and pass `?target=` on every call. Stale-target GC would prevent the trap.
6. Minor: one transient `<Image> is missing alt` warning at mount that no DOM
   img actually exhibits (all 4,894 catalog images carry alt); and `input`
   `422`s if typed into during a settle window (retry fixed it).

## What was hard

Writing ~600 lines of Contract **blind** — no LSP, no incremental compiler
feedback until the dev server reloads. The grammar itself is pleasant
(indent-structured view, `when/else`, `each key=`), and it compiled with zero
grammar errors on first load, but that felt like luck plus very conservative
use of only documented constructs. The compiler being embedded in Vite means
the feedback loop is "save, wait for reload, read the browser": fine at 60
lines, slow at 600.

Styling within the restriction list (LLP 0004) was manageable for this brand —
Brooks is square-cornered and flat, which Contract's attribute set covers. No
`position: sticky` cost the sticky add-to-bag bar; no text strikethrough cost
the struck-through out-of-stock sizes (gray boxes instead); no animation cost
the entrance/press motion. The gap vs the Expo build is exactly the *feel*
layer, as LLP 0004 predicted.

## Expo and Exact comparison

Direct, same-night, same-feature comparison (see `2026-07-13-expo-screens.md`):

- **Data/logic: near parity.** The catalog helpers ported with mechanical
  changes; the cart logic is line-for-line comparable. Contract's
  declared-writes actions caught one bug at compile time that React would have
  let slide.
- **Structure: Contract is more legible.** One file where state, actions,
  checkable contract, and view sit together reads better than seven route
  files; testIds double as the contract's own assertions.
- **Feel: not close.** Haptics, the press-shadow, swipe-to-delete with undo,
  parallax, blur, the size-grid shake — all absent from the Exact build with
  no path to them today. `temporal()` was the one delightful exception.
- **Verification: Exact wins.** Acto's snapshot/tap/testId loop beats
  Playwright text-selectors against RN-web (which kept matching hidden
  screens). The contract block plus agent API is a genuinely better
  agent-development story.

## Improvement ideas

- **Exact:** fix the browser agent bootstrap (guard the code-mode import, or
  lazy-load it server-side only). It defeats the scaffold's own headline
  feature — the shims in this repo show the minimal viable fix.
- **Exact:** make `exact new`'s output typecheck cleanly, initialize
  `vendor/ibex` automatically (or fail with the command to run), and name the
  CLI crate `exact` or document `-p exact-cli`.
- **Exact:** garbage-collect dead agent targets, or expose "route to the
  target this page owns" as a first-class header.
- **Exact:** a `exact contract check --watch` CLI loop (compile errors without
  the dev-server round trip) would transform long-file authoring for agents.
- **Docs:** LLP 0004 should be amended: `temporal()` exists and works
  (countdowns are possible), and the web agent bootstrap gotcha now has a
  deeper root cause than "add ensureAppAgentBootstrap()".
